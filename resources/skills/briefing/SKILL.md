# 每日简报生成

从 `./articles/` 的文章生成一份**有价值、可快速读完**的简报，写入 `./briefings/<日期>.md`。

## 输入

- 文章在 `./articles/<sourceType>/<sourceId>/<id>.md`（frontmatter + 正文）。
- 默认范围：**最近 1-2 天**发布的文章（看 frontmatter 的 `publishedAt`，UTC 毫秒）。
  用户若指定范围/日期，按用户要求。
- 若文章已有 `summary`/`score`（summarize 技能产出），优先用它们，省重复阅读。

## 处理流程

1. **筛选**：按 `publishedAt` 取范围内文章。若数量 > 30，先按下述价值排序取 Top 20-30（避免过长、避免 lost-in-the-middle）。
2. **价值排序**：综合判断，高到低：
   - 时效（越新越靠前）、信息量（有具体事实/数据/结论）、重要性（影响面）。
   - **去重聚合**：同一事件多篇只留最全的一篇作代表，其余在该条下标"另见：<来源>"。
   - 软文/纯广告/标题党压到最低或剔除。
3. **分类**：按主题归入板块（如「大模型 / 产品 / 研究 / 工具 / 其他」，按当天实际内容定）。
4. **写每条**：标题 + 2-3 句摘要（带具体实体，不写废话）+ **一句「为什么重要」**（这是简报价值核心）+ 来源与原文链接。
5. **写总览**：开头一段 TL;DR，点出当天最值得知道的 2-3 件事。

## 输出格式（写入 `./briefings/<YYYY-MM-DD>.md`）

```markdown
# 每日简报 · <日期>
> 约 N 分钟读完 · 覆盖 X 篇 · 精选 Y 条

## 总览
<一段 TL;DR：今天最重要的 2-3 件事>

## <分类名>
### <文章标题>
<2-3 句摘要>
**为什么重要**：<一句解读>
· 来源：<公众号/站点名> · [原文](<sourceUrl>)
<若有合并：>另见：<其他来源名>

（更多条目…）
```

## 要求

- **只依据文章内容**，不臆造；信息不足就如实简短。
- 头条位（第一条）留给唯一最重要的一条。
- 中文输出，简洁克制。
- 只读 `./articles`、只写 `./briefings`，不碰 `secrets/`。
- 文件名用日期；同日多次生成则覆盖当天文件。
