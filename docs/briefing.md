# 每日简报（P2）设计

> 上级：[overview.md](overview.md) · AI 底座见 [agent.md](agent.md) · 处理层见 [process.md](process.md)
> 状态：调研完成、方案定稿，**待用户确认后动手**。

职责：从已采集的一批文章里，用 Claude 生成一份「每日简报」——总览 + 按价值排序的条目（摘要 + 为什么重要 + 原文链接）。

## 一、技术底座（实测定稿）

**用 spawn `claude -p --output-format json`，cwd 指向数据目录。** 见 [agent.md 实测定稿](agent.md#-实测定稿2026-07-06以此为准)。

- ✅ 复用本机 CLI 登录态，**用户零配置、不用配 API key**（实测：未设 ANTHROPIC_API_KEY 照跑）。
- ✅ claude 自主 Read 数据目录的 markdown，`.result` 即最终结果。
- ✅ 返回 JSON 含 `total_cost_usd`，成本可展示给用户。
- 不走 Agent SDK（要配 key、按 token 计费）；不走流式（简报是一次性产物，取 `.result` 即可）。

## 二、方案分层：MVP 先行，别过度设计

调研给的"严肃系统"方案（embedding 去重 + HDBSCAN 聚类 + 微调模型查标题党 + map-reduce）**对当前阶段过重**。按"够用即可"裁剪成两档：

### MVP（先做这个）—— 让 Claude 一步到位

几十篇文章、每篇几千字，**正文已在本地 markdown**。直接把「文章清单 + 正文」交给 claude，让它在一次 agentic 调用里完成：读取 → 判断价值 → 排序 → 摘要 → 产出简报。

理由：
- claude 上下文足够（几十篇 ~几万 token，Opus 200k 窗口够）。
- claude 本身就能做"价值判断/去重/摘要"，不必自己先搭 embedding+聚类那套重机器。
- 一步到位、代码最少、最快能用。**先跑起来看效果，不够再加。**

⚠️ 注意调研的 **Lost in the Middle**（长上下文中间信息易被忽略）：所以 MVP 阶段限制**单次简报最多约 30-40 篇**；超了先按时间截断，或分批。真到几百篇再上 map-reduce。

### 演进（不够时再加，按需）

- **廉价预排序**：进 claude 前先用时间衰减 + 来源权威 + 简单去重砍到 Top-N（省 token、避开 lost-in-the-middle）。
- **语义去重聚类**：同事件多篇合并（embedding + 阈值/HDBSCAN）——真出现大量重复再做。
- **软文/标题党过滤**：清洗阶段剥公众号页尾 CTA（点赞/在看/二维码）、识别软广——先靠 claude prompt 判断，不够再上专门模型。

## 三、简报内容结构（采纳调研的收敛模板）

```
# 每日简报 · <日期>
> N 分钟读完 · 共 X 篇 · 精选 Y 条

## 总览
<一段 TL;DR，今天最值得知道的 2-3 件事>

## <分类：如 大模型 / 产品 / 研究 / 工具>
### <标题>
<2-3 句摘要>
**为什么重要**：<1 句解读>
· 来源：<公众号名> · [原文](url)
...
```

关键点（调研确证）：
- **「为什么重要」这层**是把"新闻搬运"变成"有价值简报"的签名特征（Import AI 范本）。
- 头条位留给唯一最重要的一条；其余按价值降序。
- 摘要用 **Chain-of-Density 思路**：强制每句带具体实体（Relevant/Specific/Novel/Faithful），避免"正确的废话"。
- **抗幻觉**（Anthropic 官方）：prompt 里要求"只依据所给正文、可说不知道、结论需有原文支撑"。

## 四、打分维度（MVP 让 claude 综合判断，prompt 里给标准）

不搭独立打分系统，在 prompt 里让 claude 按这些维度综合评分（1-10）：
- **时效**：越新越高。
- **信息量**：有具体事实/数据/结论 vs 空泛。
- **重要性**：影响面、是否重大事件。
- **去重**：同一事件多篇只保留最全的一篇，其余标"另见"。
- **过滤**：软文/纯广告/标题党（点开无实质）压低分或剔除。

## 五、数据流与产物

```
用户点"生成简报"（选日期/范围）
  → 收集范围内 Article（store 已有；正文在 md 文件）
  → 组装 prompt（清单 + 正文，或让 claude 自己读 data/articles/）
  → spawn claude -p --output-format json（cwd=data/）
  → 拿到结构化简报（建议让 claude 输出 JSON：{overview, sections:[{category, items:[...]}]}）
  → 存为 data/briefings/<date>.md（文件为源，可回看）+ 索引
  → UI 展示（新增"简报"入口/视图）
```

产物也落文件（`data/briefings/`），符合"数据即文件"，通用 agent 可直接读。
