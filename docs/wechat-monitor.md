# 公众号/RSS 阅读器：当前产品形态

> 上级：[overview.md](overview.md) · 登录与限流：[wechat-login.md](wechat-login.md) · 采集接口：[ingest.md](ingest.md)

文件名保留 `wechat-monitor.md` 以兼容既有链接。当前产品支持手动采集和用户显式开启的定时采集；自动采集默认关闭、最短周期 1 小时，且始终受账号小时上限、全局串行和请求间隔保护。

## 当前交互

```text
扫码登录公众号后台账号（RSS 不需要）
        │
        ▼
添加信源：公众号名称搜索 / RSS URL 解析
        │
        ├─ Source 立即写入 sources.json
        └─ 后台触发首次采集
                │
                ▼
三栏阅读：信源列表 │ 文章流 │ 沉浸阅读 / 原始排版
                │
                ├─ 拉取当前源或全部启用源的最新列表（不回溯）
                ├─ 可选按 1 小时～7 天自动拉取全部已启用信源
                ├─ 重抓本篇 / 按单源或全部处理已入库历史
                ├─ 打开即标记已读 / 当前范围一键全部已读
                └─ 归档 / 打开原文
```

## 阅读界面与设置

### 顶部工具栏与三栏布局

- 顶部保留产品标识、三栏显隐、“来源与抓取”和设置入口。当前 Logo 暂不继续裁剪；等待用户提供成品后再替换。
- 全局提供白色与暗色两套主题、扁平边界和青绿色强调；偏好只保存在当前设备。
- 信源、文章、正文三栏都可独立隐藏；即使全部隐藏，也能从工具栏或空白页恢复默认布局。
- 相邻栏分隔线支持拖动调宽、方向键微调和双击恢复；宽度与显隐只持久化到本机界面偏好。
- 文章栏标题过长时省略显示，header 不参与内容区收缩，“我的 / 团队”范围切换始终保持在文章列表上方。

### 左栏：信源

- “全部”聚合入口、每源未读数、来源类型、暂停状态和账号告警点。
- “拉取全部最新”只处理启用来源；添加公众号/RSS 继续使用独立弹窗。

当前限制：

- 后端支持删除账号，UI 没有删除入口。
- 微信信源告警仍被简化成“没有 active 账号”，尚未细分每个公众号的实际可抓取状态。

### 设置页面

- “来源与抓取”：把“检查新文章”和“修复已保存文章”作为两个独立动作；单源维护默认折叠，全量维护独立放在高级区域，取消关注明确展示删除影响。
- “账号与自动化”：扫码登录、失效账号重登、每账号小时上限、自动采集周期和运行状态。
- “数据资料库”：查看、打开或迁移当前内容资料库，并显示外部 `outputs/` 路径和上次迁移结果。
- “团队共享”：加入/退出团队、连接身份、服务器地址、待上传/隔离数、上次/下次同步、自动同步开关、1～1440 分钟周期和立即同步。
- “阅读界面”：选择白色/暗色主题、全局正文显示习惯，并恢复默认三栏布局。
- “软件更新”：手动检查、下载状态和重启安装；原生菜单仍作为 renderer 故障时的恢复入口。

### 中栏：文章流

- 当前源或全部文章，按发布时间倒序，最多 500 条。
- 全部 / 未读 / 已归档筛选。
- 未读点、标题、来源、相对时间。
- “拉取最新”明确不回溯未入库历史；采集和历史任务使用不同进度文案。
- “全部已读”按当前来源和“我的 / 团队”范围处理所有未归档文章，不受可见列表 500 条上限影响。

自动采集的启用状态、下次运行时间、最近结果和错误通过独立 `collection-status` 事件展示。`waiting_quota` / `waitingUntil` 仍只是采集进度契约留位，当前不会跨时间排队等待账号恢复。

### 右栏：详情

- 标题、作者、来源、发布时间、正文。
- 查看原文、归档，以及只联网处理当前文章的“重抓本篇”。
- 微信和 RSS 可使用窄行宽、衬线字体的 Markdown 沉浸阅读；微信有正文 HTML 时可切换 iframe 原始排版。选择会跨文章保留，无原始 HTML 时只临时回退沉浸阅读。
- iframe 隔离公众号 CSS 与滚动，不承担恶意内容安全模型；官方 `mp.weixin.qq.com` 正文按产品决定视为可信来源。
- 原始排版保留公众号自己的浅色与内联颜色，不强制套用 App 暗色主题；正文链接统一由系统默认浏览器打开，不提供内置浏览器弹窗。
- Markdown/RSS 仍保留 DOMPurify、URL allowlist、CSP 和公众号图片 Referer 处理。
- `article:list` 只查询 SQLite 摘要；`article:get` 读取单篇 Markdown；只有实际进入原始排版时才读取 `.content.html` sidecar。

依赖微信页面脚本、完整运行时或鉴权的动态组件可能无法从静态正文恢复，遇到此类内容使用“打开原文”。

当前没有摘要、分数、标签、简报、反归档或手动改未读入口。

## 添加信源

### 公众号

```text
输入名称
  → WechatAdapter.discover() 调 searchbiz
  → 展示候选 nickname / alias / signature / avatar
  → 选择候选
  → Source{id: wx-<fakeid>, type: wechat, config:{fakeid,...}}
  → 后台采集一页（最多 10 条）并补正文
```

需要至少一个可用账号。搜索和文章列表请求都计入保守小时配额。

### RSS

```text
粘贴 http(s) feed URL
  → RssAdapter.discover() 拉取并解析
  → Source{id: rss-<urlHash>, type: rss, config:{feedUrl}}
  → 后台采集 feed 条目
```

RSS 不需要账号或微信限流，但当前仍与微信任务共用 Collector 全局串行锁。

## 手动与自动采集模型

- 添加成功后，IPC 立即返回；首次采集在 main 后台继续。
- 拉取单源或全部源时，Service 逐源调用 Collector；全部范围只选择 `enabled` 来源，显式单源手动拉取可处理暂停来源。
- 自动采集默认关闭；用户可选择每 1/2/4/8/12 小时、每天或每 7 天运行，修改后从当前时间重新等待一个完整周期。
- 自动轮次只处理 `enabled` 信源，按最久未采集优先；手动、首次和自动批次共用互斥，不重叠也不积压。
- 应用未运行或电脑睡眠时不会采集，唤醒后不补跑错过轮次，而是重新等待完整周期。
- Collector 的 Promise chain 保证所有 discover/collect 全局串行。
- 微信 adapter 默认每源只拉最新 1 页、每账号每小时 20 个请求；当前没有“翻页发现未入库历史”能力。用户可在设置中修改全局单账号上限，换号间隔仍固定保守值。
- 自动轮次会在设置中保留中文结果摘要；搜索/添加、团队、配额和更新等用户操作也会显示中文错误建议。

当前没有任务插队、跨时间持久化排队或主动保活；自动采集只在桌面 App 运行期间按保守固定周期触发。

## 验收缺口

- 扫码 → 搜号 → 关注 → 首次文章出现。
- RSS URL → 添加 → 条目出现。
- 连点拉取最新仍保持全局串行且 UI 状态正确。
- 自动采集开启/关闭、周期修改、睡眠恢复和退出时中止后续批次。
- 重抓本篇、按单源/全部离线重解析与联网重抓，以及失败页不覆盖既有完整正文。
- 账号过期、限流、无账号、网络失败的 UI 提示。
- 单篇/一键已读与归档跨重启、范围语义和 v0.1.0 旧数据迁移（已有自动化测试，仍需桌面人工复验）。
- sandbox preload、CSP 下公众号图片、外链系统浏览器行为。
- 微信默认沉浸阅读、原始排版切换、长文滚动、iframe 中懒加载图片和动态组件降级。
- 取关后的文章、原始载荷和当前选中状态。
- 高 DPI、白色/暗色主题和 Windows 下拖动/显隐的人工验收；布局边界与损坏偏好回退已有自动化测试。
