# 公众号/RSS 阅读器：当前产品形态

> 上级：[overview.md](overview.md) · 登录与限流：[wechat-login.md](wechat-login.md) · 采集接口：[ingest.md](ingest.md)

文件名保留 `wechat-monitor.md` 以兼容既有链接，但当前产品并不是后台自动监控器：**没有定时轮询，只在用户主动添加或刷新时采集。** 这是保护真实公众号账号的硬约束。

## 当前交互

```text
扫码登录公众号后台账号（RSS 不需要）
        │
        ▼
添加信源：公众号名称搜索 / RSS URL 解析
        │
        ├─ Source 立即写入 sources.json
        └─ 后台触发首次采集
                │
                ▼
三栏阅读：信源列表 │ 文章流 │ Markdown 正文
                │
                ├─ 手动刷新当前源或全部
                ├─ 打开即标记已读
                └─ 归档 / 打开原文
```

## 三栏 UI 实现

### 左栏：信源与账号池

- “全部”聚合入口、每源未读数和状态点。
- 刷新全部、添加公众号/RSS 弹窗。
- 账号数量、可编辑的每账号小时上限、active/cooldown/expired 状态。
- 本小时与累计请求数，以及最近一次 `200013` 的请求序号和时间。
- 扫码登录新账号、失效账号重新登录。

当前限制：

- 按钮仍显示“+ 加公众号”，虽然弹窗已支持 RSS。
- 后端支持删除账号，UI 没有删除入口。
- 信源告警被简化成“没有 active 账号”；RSS 不需要账号却也可能显示告警。
- 没有单个信源的删除/设置按钮可见入口（后端 IPC 已有 remove）。

### 中栏：文章流

- 当前源或全部文章，按发布时间倒序，最多 500 条。
- 全部 / 未读 / 已归档筛选。
- 未读点、标题、来源、相对时间。
- 手动刷新与采集中的简易进度提示。

`waiting_quota`、`waitingUntil`、`nextRunAt` 已在共享契约和 UI 留位，但 main 当前从不发送这些状态；不要据此认为排队/预计恢复已完成。

### 右栏：详情

- 标题、作者、来源、发布时间、正文。
- 打开原文、归档。
- 轻量 Markdown 渲染、DOMPurify/CSP 安全清洗和公众号图片 Referer 处理。

当前没有摘要、分数、标签、简报、反归档或手动改未读入口。

## 添加信源

### 公众号

```text
输入名称
  → WechatAdapter.discover() 调 searchbiz
  → 展示候选 nickname / alias / signature / avatar
  → 选择候选
  → Source{id: wx-<fakeid>, type: wechat, config:{fakeid,...}}
  → 后台采集一页（最多 10 条）并补正文
```

需要至少一个可用账号。搜索和文章列表请求都计入保守小时配额。

### RSS

```text
粘贴 http(s) feed URL
  → RssAdapter.discover() 拉取并解析
  → Source{id: rss-<urlHash>, type: rss, config:{feedUrl}}
  → 后台采集 feed 条目
```

RSS 不需要账号或微信限流，但当前仍与微信任务共用 Collector 全局串行锁。

## 手动采集模型

- 添加成功后，IPC 立即返回；首次采集在 main 后台继续。
- 刷新单源或全部源时，Service 逐源调用 Collector。
- Collector 的 Promise chain 保证所有 discover/collect 全局串行。
- 微信 adapter 默认每源 1 页、每账号每小时 20 个请求；用户可在左栏修改全局单账号上限，翻页与换号间隔仍固定保守值。
- 采集结果只在有新文章时触发文章刷新；错误主要写主进程控制台，UI 提示不完整。

“自动监控、每 2–4 小时轮询、任务插队、跨时间排队、主动保活”均是旧设计，没有当前实现。若未来恢复，必须默认关闭、由用户显式开启，并先完成配额估算、任务持久化、取消、错误反馈和真实账号安全验收。

## 验收缺口

- 扫码 → 搜号 → 关注 → 首次文章出现。
- RSS URL → 添加 → 条目出现。
- 连点刷新仍保持全局串行且 UI 状态正确。
- 账号过期、限流、无账号、网络失败的 UI 提示。
- 已读/归档跨重启与 v0.1.0 旧数据迁移（已有自动化测试，仍需桌面人工复验）。
- sandbox preload、CSP 下公众号图片、外链系统浏览器行为。
- 取关后的文章、原始载荷和当前选中状态。
- 窄窗口/高 DPI/深色模式下三栏可用性。
