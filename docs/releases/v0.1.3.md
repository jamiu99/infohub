# infohub v0.1.3

发布日期：2026-07-13

这是桌面 IPC 桥和公众号二维码登录修复版本。`v0.1.2` 已撤回，用户应从 `v0.1.1` 直接升级到本版本。

## 关键修复

- 修复 sandboxed renderer 无法加载 ESM preload，导致 `window.api` 缺失、扫码等按钮报错的问题。
- preload 改为单文件 CommonJS `out/preload/index.cjs`，保留 renderer sandbox 与 context isolation。
- 新增构建产物契约，阻止 ESM preload 或错误文件路径进入安装包。
- 新增真实 Electron smoke test：启动隐藏窗口，通过 `window.api.account.list()` 验证 preload 和 IPC 后自动退出。
- 微信公众平台优先显示账号/快捷登录时，自动切换到传统二维码；明确提示不要输入账号、密码或验证码。
- 登录窗口拒绝设备权限、站外导航和页面弹窗。

## 升级说明

- `v0.1.1` 会通过自动更新直接发现 `v0.1.3`。
- 请勿手动安装已撤回的 `v0.1.2`。
- 本版本仍会在扫码成功后把 Cookie/token 加密保存在本机，这是公众号采集所需的会话凭据。
- Windows 安装包尚未代码签名，SmartScreen 可能显示“未知发行者”。

## 待验收

- 自动化已验证真实 Electron preload + IPC；仍需在真实 Windows 环境确认扫码、保存账号和首次公众号采集完整通过。
