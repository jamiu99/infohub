# infohub v0.3.0

发布日期：2026-07-15

这是一次微信公众号正文归档与展示更新。经典图文不再只依赖 Markdown，而是保留 Markdown、正文 HTML 与完整页面 HTML 三份互补产物；团队同步协议也开始直接传输正文 HTML。

本版本不增加任何 AI 直接集成。

## 微信正文归档与展示

- 经典微信公众号图文使用 `parse5` 构建 HTML 树并定位 `#js_content`，解决正则提取在深层嵌套正文中提前截断的问题。
- 每篇成功获取的经典图文保存三份产物：
  - Article Markdown：稳定文本投影、检索入口和“阅读版”。
  - 同名 `.content.html`：保留正文节点顺序、外层 `#js_content` 与内联样式，供原始排版展示。
  - SHA-256 命名的 `.page.html`：完整、未改写的 HTTP 页面，供诊断和未来离线重解析。
- 微信详情有正文 HTML 时默认显示“原始排版”，可随时切换到 Markdown“阅读版”。正文 iframe 隔离公众号 CSS、尺寸与滚动，不主动运行完整微信页面脚本。
- 离线展示会主动解除微信运行时留下的正文隐藏态；只有空壳 `#js_content` 时不再误判完成，而是保留页面并等待下次刷新重试。
- `article:list` 继续返回轻量 Article；选中文章后的 `article:get` 才读取正文 HTML，避免列表 IPC 搬运大文本。

## 正文生命周期与重试

- Article frontmatter 新增正文状态、解析器版本、正文/页面相对路径、最后尝试时间、最后成功时间和中文错误信息。
- `seen_items` 只负责条目去重，不再把“已经见过”误当成“正文已经完整”。
- 当正文失败或不完整、解析器版本落后、`.content.html` 丢失，或微信文章缺少本机 `.page.html` 时，只要条目再次出现在手动刷新结果中就会重试。
- 重试失败不会以空正文覆盖已经成功保存的 Markdown 或正文 HTML。HTTP 已返回页面但未找到正文时，仍会保留完整页面供分析。

## 团队同步协议：破坏性变更

- 所有团队端点从 `/api/v1` 硬切到 `/api/v2`；客户端不请求 v1、不在 404 后 fallback。同步先检查 `/api/v2/status`，误连旧服务时会保留 outbox 并暂停，不会先发送新 DTO。
- Article 有正文 HTML 时，桌面端会把 `contentHtml` 直接放入团队同步 DTO；不做能力协商。
- 团队同步内容为 Source 白名单字段、Markdown、正文 HTML、原文 URL 与公开 Article 元数据。
- 完整 `.page.html`、列表 Raw、Cookie、后台 token、fingerprint、浏览器 session、本地路径和正文诊断状态仍不上传。
- Markdown 单篇上限为 2 MiB，正文 HTML 单篇上限为 4 MiB；超限或不符合 allowlist 的事件进入本机隔离区，不持续阻塞其他队列。
- push 按 `JSON.stringify({items})` 的实际 UTF-8 字节数分批，客户端预算 12 MiB、服务端上限 16 MiB；pull 每页最多 50 条。单项因 JSON 转义后超过预算时会在本地直接隔离并显示原因。
- 确定性 eventId 的协议版本输入已改为 2；旧 outbox/ack/quarantine 不迁移，也不会阻止当前协议重新生成事件。
- 阅读与归档只改变本地状态，不再推进内容 `updatedAt`，避免历史补传把旧正文伪装成团队新版。

本协议不兼容旧桌面端或旧服务端，也不支持混合版本运行。升级顺序必须是：

1. 暂停旧桌面端团队同步。
2. 先升级 `infohub-team-server` 并确认健康检查正常。
3. 再升级全部桌面端到 `v0.3.0`。
4. 恢复同步并验证两台设备可以互相取得 `.content.html`。

App 内自动更新只能更新桌面端，不能升级自托管团队服务。使用团队功能的用户必须先完成服务端升级，再确认桌面端更新。

## 数据与迁移

- SQLite schema 仍为 v3，不需要数据库迁移；HTML 作为普通 sidecar 文件保存。
- 已有 Markdown 可以继续读取。旧文章没有正文状态或 HTML sidecar 时，会在它再次出现在手动刷新列表后尝试补齐三份产物。
- 已存在的团队 outbox、ack、quarantine、cursor 和设备凭据仍保存在原目录；不迁移旧标记，也不承诺旧协议队列与混合版本服务互操作。API v2 eventId 使用新版本输入，当前事件不会被旧 ack/quarantine 拦截。
- 完整页面 HTML 只保存在真实抓取它的本机，不会通过团队同步补到其他设备。

## 验证基线

- `pnpm typecheck` 通过。
- `pnpm test:core`：80/80 通过。
- `pnpm build` 与 sandbox preload bundle 检查通过。
- Linux Electron smoke test 已验证 preload/IPC，以及 CSP 下 `srcdoc` iframe 和微信官方 base URL。

## 当前限制

- 新版/旧版 SSR 图片页尚未形成完整正文 HTML；风控页、验证页和页面结构变化仍可能只保存完整页面并标记失败。
- 依赖微信运行时脚本的音视频、卡片、互动排版等动态组件可能无法在静态正文中完整恢复，可使用“打开原文”。
- 还没有一次性重解析所有历史 `.page.html` 的入口；当前补齐依赖条目再次出现在手动刷新结果中。
- 图片和其他媒体仍引用微信原始 URL，不下载为本地二进制，也不通过团队同步。
- 尚未完成真实 Windows 的安装、`v0.2.1` → `v0.3.0` 自动更新和两设备团队 HTML 同步人工验收。
- Windows 安装包暂未代码签名，首次运行可能显示“未知发行者”。
