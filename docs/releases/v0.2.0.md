# infohub v0.2.0

发布日期：2026-07-13

这是首个团队共享版本。infohub“文件为源、SQLite 为可重建索引”的原则不变；团队功能是可选的，自托管后端保持为独立项目，不引入任何 AI 集成。

## 新增

- 支持通过 HTTPS 团队地址、共享 `TEAM_TOKEN`、成员名和设备名首次加入团队。
- 设备 token 由服务端签发，客户端不自行生成；`TEAM_TOKEN` 请求结束后不保存。
- 本机已有和新采集的允许共享数据默认进入文件型 outbox；网络失败不影响本地落盘和采集。
- eventId 确定性派生，服务端成功后本机写 ack 标记；首次历史补传可在重启后恢复且不会反复上传。
- 超限、非法或被服务端永久拒绝的单篇事件会隔离，其余文章继续同步，侧栏显示隔离数量。
- 支持按 cursor 增量 push/pull，并在看板切换“我的 / 团队”。
- 团队文章继续保存为普通 Article Markdown，SQLite schema v3 增加 `contributed_by_me` 派生索引。
- 取消本地订阅时保留已同步团队副本和个人阅读/归档状态；重新订阅不会复制 Article。
- 自托管 `infohub-team-server` 提供单实例单团队、Source/Article 去重、贡献记录、来源分配和短租约 API，默认端口 `18038`。

## 安全边界

- 正式客户端只接受 HTTPS 团队地址，默认值为 `https://home.agent-wiki.cn:18038`。
- 只上传明确 allowlist 的 Source 元数据和归一化 Article。
- Raw、微信 Cookie、后台 token、fingerprint、Electron partition/session 永不上传。
- 团队设备 token 必须使用 Electron `safeStorage` 加密；系统安全存储不可用时拒绝持久化并让入组失败，不回退明文。
- 私有 RSS URL 中的账号密码，以及名称疑似 token、secret、password、API key 或签名的查询参数不会上传。

## 数据与迁移

- 原有 Article、Raw、`sources.json`、阅读/归档状态和账号数据保留。
- Article frontmatter 新增可选 `team` 字段；旧文件没有该字段时仍视为本机历史贡献。
- `index.sqlite` 自动升级到 schema v3；索引仍可从 Article 文件完整重建。
- 新增 `data/team/outbox/`、`acked/`、`quarantine/`、`sync-state.json` 和 `data/secrets/team-device.enc`。前四者是同步状态，后者是敏感设备凭据，都不是外部内容接口。

## 当前限制

- 默认团队地址只有在后端和可信 TLS/FRP 入口部署完成后才能连接。
- 来源分配、轮询均分和租约已提供服务端 API；桌面端“我的采集任务”管理界面尚未实现。
- 隔离事件目前只显示数量，尚无详情、删除或修复后重试界面。
- 尚未完成两台 Windows 设备的真实端到端同步和从 v0.1.5 自动更新人工验收。
- 服务端 MVP 按团队成员互相信任设计；细粒度权限、审计、限时邀请和端到端加密是极低优先级未来能力。
