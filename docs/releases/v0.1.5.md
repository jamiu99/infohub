# infohub v0.1.5

发布日期：2026-07-13

这是采集配额可配置与测试期限流观测版本。

## 配额设置

- 左栏新增“每账号每小时上限”，默认仍为保守的 20，可配置 1–1000，保存后立即生效。
- 所有微信账号共用这个上限，但各账号仍独立计算本小时已用次数。
- 设置写入非敏感 `data/settings.json`；Cookie/token 仍只在账号凭据文件中。
- 调低上限不会清空本小时已用次数，避免通过反复改配置绕过保护。
- 超过 50 时看板提示风控风险；50 只是历史观察值，不是微信的稳定承诺。

## 测试期限流观测

- 每个账号显示本小时请求数和本机累计请求数。
- 命中 `200013` 后记录“本小时第几次、累计第几次、发生时间”，重启和小时窗口重置后保留。
- 触发限流的那次请求已经包含在计数内；统计范围是需要登录态的 `searchbiz` / `appmsg` 请求。
- 只做被动记录，不会自动压测、制造并发或主动撞限流。

## 数据兼容

- 不改变 Article、Raw 或 SQLite 内容接口。
- 旧账号记录缺少累计值时，以升级当时的本小时请求数作为累计起点；Cookie/token 和独立 session 分区不变，无需重新登录。
- 配置文件缺失、损坏或越界时安全回退默认 20。

## 验证与限制

- 49 项核心测试、生产构建、sandbox CommonJS preload 契约均通过。
- 真实 Electron smoke test 已验证账号列表、设置持久化 IPC 与更新菜单。
- Windows 安装包仍未代码签名，SmartScreen 可能显示“未知发行者”。
- 上游接口和风控阈值可能随账号与时间变化，请只采集自己有权限的数据并逐步调整。
