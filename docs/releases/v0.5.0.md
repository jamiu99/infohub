# infohub v0.5.0

发布日期：2026-07-15

这是阅读体验与来源控制版本。它在不改变资料库 schema 和团队同步协议的前提下，重新整理阅读界面、抓取动作与按来源管理，让用户能明确知道一次操作会处理哪篇文章、哪个来源以及是否访问网络。

## 阅读体验

- 主界面升级为米色纸张色阶与扁平化边界，减少阴影和装饰，强化文章列表与正文层级。
- 微信文章默认使用窄行宽、中文衬线字体和舒展行距的 Markdown 沉浸阅读；仍可切换公众号原始排版。
- 顶部复用既有透明 Logo，并通过界面裁切只展示中心图形；Windows 安装包图标素材保持不变。
- 当前来源与“我的 / 团队”范围新增一键全部已读；处理后台全部未归档文章，不受看板 500 条列表上限影响。

## 来源与抓取

- 设置新增独立“来源与抓取”页面，可逐个微信公众号或 RSS 启用/暂停、手动拉取最新、处理历史和取消关注。
- “重抓本篇”只联网访问当前文章 URL，不会处理同来源的其他文章。
- “拉取最新”可以处理当前来源或全部启用来源；微信公众号仍默认只取最新一页，不回溯未入库旧文章。
- “处理已入库历史”可以按单来源或全部来源运行，也可选择离线重解析已有页面快照或联网重抓原文；它包含归档文章，但不会发现从未入库的文章。
- 全部来源联网重抓已入库历史需要额外确认，降低误触造成大量网络请求的风险。
- 暂停来源只会把它排除在自动采集和“全部来源拉取最新”之外，仍允许用户显式执行单来源操作。

## 数据与兼容性

- 新增来源启停与批量已读 IPC；批量已读沿用既有 Article 状态字段并逐篇写回文件，使 SQLite 重建后状态仍然保留。
- 没有资料库 schema、Article 正文格式或团队 `/api/v2` 协议变化，从 `v0.4.2` 可直接升级。
- 阅读/归档状态仍是纯本机状态，不推进内容 `updatedAt`，也不会制造新的团队同步事件。

## 验证

- TypeScript 与 Vue 类型检查通过。
- 153 项核心测试全部通过，新增覆盖单来源、我的/团队范围、归档排除和文件重建后的批量已读状态。
- Electron main、preload 与 renderer 生产构建及桌面 bundle 契约通过。

## 已知限制

- 本版没有微信公众号任意时间范围的历史发现 API；“拉取最新”只读当前列表，“处理已入库历史”只处理本地已经存在的 Article。
- 旧 `__QMTPL_SSR_DATA__`、依赖脚本的动态组件、表格/音视频/卡片语义投影仍未完整支持，必要时需打开原文。
- npm registry 审计端点在发布时返回 410，仓库 Dependabot 也未启用，因此本版没有新的自动化依赖漏洞审计结论。
- Windows 安装包尚无代码签名，SmartScreen 可能显示未知发行者。
- 真实 Windows 安装、从 `v0.4.2` 跨版本自动更新、扫码与长文阅读仍需人工验收，不能仅以自动化门禁替代。
