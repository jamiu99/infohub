# infohub v0.4.0

发布日期：2026-07-15

这是数据生命周期版本：infohub 仍然不直接集成 AI，但资料库现在可以由用户选择、迁移并交给外部工具只读消费；原始证据、产品正文和外部处理结果有了明确边界。

## 新功能

- 支持微信公众号图片消息/“贴图号”：读取 `picture_page_info_list` 的正文图片和图注，生成可展示 HTML 与 Markdown，不执行页面脚本。
- 文章详情可联网“重新抓取”；设置中可按当前信源或全部本机文章执行离线重新解析或联网重抓，包含归档文章。
- 自动采集可由用户显式开启，周期为 1 小时～7 天、推荐 4 小时；默认关闭，休眠/退出期间不追赶错过轮次。
- 设置新增“数据资料库”：显示并打开当前目录，可迁移到用户选择的空目录。
- 资料库新增 `outputs/<producer>/`，作为 AI Agent、脚本等外部处理程序唯一允许写入的区域。
- Windows 应用 EXE、桌面/开始菜单快捷方式、安装器和卸载器继续统一使用 infohub 项目 Logo，不再出现 Electron 默认图标。

## 数据与安全边界

- Raw 列表响应和微信完整页面改为按内容 SHA-256 寻址，只创建、不覆盖；不同响应永久并存。
- Article/`.content.html` 是 infohub 管理的当前投影。失败重抓只记录最新错误与响应快照，不覆盖已有完整正文或成功页面。
- 离线重新解析只读取本机快照；联网重抓才访问原文。微信后台请求保持 10 秒全局间隔、换号 15 秒，公开文章页按 2 秒串行。
- 外部消费者只读 `articles/`，不得修改 `articles/` 或 `raw/`；派生结果写 `outputs/<producer>/`，infohub 不回灌这些结果。

## 升级与迁移

- 从 v0.3.x 首次启动时，旧数据目录中的 `settings.json`、`secrets/`、`team/` 会复制到 Electron 固定私有 `state/`；已有目标绝不覆盖，旧文件不会删除。
- 用户主动迁移资料库时，应用会安全关闭并重启，在目标盘 staging 逐文件复制和 SHA-256 校验；全部通过后才切换到新位置。
- 迁移目标必须为空目录；源资料库始终完整保留，需要用户确认新目录稳定后自行处理。
- 账号 Cookie、团队 token、设置和 Chromium 登录分区属于本机私有状态，不随资料库迁移。
- 团队 API 仍为 v0.3.0 引入的 `/api/v2`，本版没有新的服务端协议升级。

## 已知限制

- 已覆盖当前样例图片消息，但旧 `__QMTPL_SSR_DATA__`、依赖脚本的动态组件、表格/音视频/卡片语义投影仍未完整支持；无法静态恢复时保留“打开原文”。
- 当前不下载或团队同步图片二进制，正文仍引用原始媒体 URL。
- 自动采集要求 infohub 正在运行；本版有意不做补抓追赶。
- Windows 安装包尚无代码签名，SmartScreen 可能显示未知发行者；真实跨盘迁移和跨版本自动更新仍需人工验收。
