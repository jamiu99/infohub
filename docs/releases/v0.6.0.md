# infohub v0.6.0

发布日期：2026-07-17

这是阅读性能与设备偏好版本。它消除了切换信源时的全资料库扫描，将公众号原始 HTML 改为按需读取，并补齐双主题、跨文章阅读习惯、来源设置说明和团队同步周期。

## 阅读性能

- `article:list` 直接从 SQLite 返回标题、来源、时间和阅读状态，不再扫描并解析最多 500 篇 Markdown。
- 未读统计、来源筛选与普通列表刷新不再触发全目录同步；短时间连续的文章变化事件会合并刷新。
- `article:get` 只读取当前文章 Markdown；只有进入“原始排版”时才读取 `.content.html` sidecar。
- 快速切换文章会丢弃已过期的详情响应，避免旧文章覆盖新选择。
- 批量已读使用批次级索引标记与单个 SQLite 事务，仍逐篇写回 Article 文件以保留可重建状态。

## 阅读界面

- 新增白色与暗色两套看板/沉浸阅读主题，偏好只保存在当前设备。
- “沉浸阅读 / 原始排版”成为跨文章习惯；没有原始 HTML 的文章只临时回退，不修改用户选择。
- 公众号原始排版保留发布者自己的浅色页面和内联颜色，不做强制反色。
- 微信正文和 Markdown 外链统一交给系统默认浏览器；子 frame 不会再把阅读区替换成无法返回的网页。
- 当前 Logo 保持不变；等待用户提供裁剪成品后再替换，作为低优先级品牌事项。

## 来源与团队设置

- “来源与抓取”明确分为“检查新文章”和“修复已保存文章”两个独立操作。
- 当前来源维护与全部来源维护分别折叠；联网影响、处理范围和删除后果使用直接中文说明。
- 团队自动同步支持独立关闭与 1～1440 分钟周期，推荐 5 分钟，并显示下一次同步时间。
- 关闭自动同步只暂停定时网络请求；新采文章继续写入 outbox，“立即同步”仍可使用。
- App 启动或重新开启自动同步时会立即同步一轮；之后在每轮完成后安排单次 timer，手动同步或修改周期会重置倒计时，且不会并发执行。

## 数据与升级

- Article Markdown、Raw、正文 sidecar 和团队 `/api/v2` 协议均未改变，不需要升级 `infohub-team-server`。
- SQLite 列表投影增加 `source_name` 与 `content_html_path`，并使用 `index_projection_version/index_dirty` 管理恢复。
- 从旧版本首次启动 `v0.6.0` 时会自动完整重建一次本地 SQLite 索引；之后正常启动和阅读查询直接复用索引。
- 旧团队设置缺少调度字段时迁移为“自动同步开启、每 5 分钟”；损坏字段只回退自身，不清空设备凭据或团队连接。
- 外部消费者仍必须只读 `articles/`/`raw/`，派生产物写入 `outputs/<producer>/`；普通 UI 查询不再为外部违规修改自动回灌索引。

## 验证

- TypeScript 与 Vue 类型检查通过。
- 166 项核心测试全部通过。
- Electron main、CommonJS preload、renderer 生产构建和 bundle 契约通过。
- 隐藏 Electron 桌面 smoke test 验证 preload、账号/采集/资料库/团队 IPC、`srcdoc` iframe 与更新菜单。

## 已知限制

- 首次升级的索引重建时间取决于资料库大小；完成后才会获得新的快速列表路径。
- 首次启动前请关闭正在读取 `index.sqlite` 的外部工具；如果提示数据库被占用，关闭工具后重启即可，Article 与 Raw 文件不受影响。
- 公众号原始排版固定浅色；暗色阅读请使用沉浸阅读。
- 自动同步仅在 App 运行时生效；设为 1 分钟会显著增加自托管服务请求量，长时间关闭则会累积待上传队列。
- 降级到不认识新调度字段的旧客户端并保存设置，可能丢失同步周期偏好；再次升级后请复核该设置。
- 本版仍不支持微信公众号任意时间范围的历史发现，旧 `__QMTPL_SSR_DATA__` 和依赖脚本的动态组件也未完整覆盖。
- Windows 安装包尚无代码签名，SmartScreen 可能显示未知发行者。
- 真实 Windows 安装及从 `v0.5.0` 的确认式自动更新仍需人工验收，自动化门禁不能替代实际点击验证。
