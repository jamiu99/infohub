# infohub v0.2.1

发布日期：2026-07-13

这是一次阅读体验与错误反馈更新。团队同步和数据文件契约保持兼容，不增加任何 AI 直接集成。

## 界面改进

- 主界面只保留信源、文章和正文阅读操作；账号配额、团队连接、界面选项和软件更新统一迁入独立设置弹窗。
- 修复长信源名称下“我的 / 团队”切换区可能被文章内容覆盖的问题；标题现在会省略显示，文章栏头部不再参与内容区收缩。
- 信源、文章、正文三栏支持拖动调宽和独立隐藏；顶部工具栏始终可以恢复隐藏栏目。
- 分隔线支持键盘方向键微调和双击恢复默认布局；栏目宽度与显隐状态在本机持久化。
- 左栏“+ 加公众号”改为“+ 添加信源”，与同时支持公众号和 RSS 的实际能力一致。

## 错误反馈

- 不再直接向用户显示 `The operation was aborted due to timeout` 一类底层英文异常。
- 超时、断网、拒绝连接、DNS、HTTPS 证书和常见 HTTP 状态会显示对应的中文原因与处理建议。
- 团队同步、文章加载、公众号/RSS 添加、账号配额和软件更新共用同一错误转换规则；原始异常仍保留在主进程日志中供诊断。

## 团队服务

- 默认团队入口 `https://home.agent-wiki.cn:18038/healthz` 已验证可以通过可信 HTTPS 返回健康状态。
- 团队入组、设备 token、outbox、增量同步和“我的 / 团队”语义与 v0.2.0 保持不变。
- Windows 正式包使用系统安全存储保存设备 token；WSL/部分 Linux 桌面若没有可用的 Electron `safeStorage`，会明确拒绝团队入组，不会回退为明文保存。

## 数据与迁移

- Article、Raw、SQLite schema、团队同步协议和账号数据格式均未变化，不需要数据迁移。
- 新增的三栏布局偏好只保存在 renderer `localStorage`，不会写入文章文件、账号凭据或团队同步数据。
- 从 v0.2.0 升级会保留原有数据、账号设置、团队 outbox 和阅读/归档状态。

## 当前限制

- 尚未完成 v0.2.0 → v0.2.1 的真实 Windows 自动更新人工验收。
- 尚未完成两台 Windows 设备的端到端团队同步验收。
- 默认团队服务仍需补进程守护、数据目录备份和外部健康监控。
- Windows 安装包暂未代码签名，首次运行可能显示“未知发行者”。
